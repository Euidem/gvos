# GVOS Stitch UI Prompt Pack

## 1. Purpose

This document contains the UI prompts to use in Stitch for designing GVOS.

GVOS is a secure operations platform for GetVirtual. The UI must feel premium, professional, secure, calm and operationally powerful.

The goal is to create screens that Claude Code can later implement with strong fidelity.

## 2. Global UI Direction for Stitch

Use this direction before generating any screen.

### Master Design Direction Prompt

Design a premium SaaS operations platform called GVOS. GVOS is used by a virtual assistant company to manage clients, business accounts, talents, line managers, workspaces, tasks, chat, files, time summaries, reports, billing, complaints and secure password sharing.

The design should feel modern, clean, secure, calm, trustworthy and enterprise ready. It should not look like a cheap admin panel. It should look like a polished operations command center used by a serious remote talent company serving US, UK and European clients.

Use a light main interface with a dark elegant sidebar. Use clean white cards, soft shadows, subtle borders, rounded corners, professional spacing, clear headings, readable typography, strong status badges and beautiful dashboard metrics.

The visual style should be premium but not flashy. It should feel like a mix of Linear, Slack, HubSpot, ClickUp and a modern fintech dashboard.

Use responsive layouts that will work well on desktop and mobile.

Core design principles:

1. Workspace first.
2. Role based dashboards.
3. Clear task tracking.
4. Secure communication.
5. Manager visibility.
6. Client confidence.
7. Talent productivity.
8. Operational control.

Do not use GetVirtual as the product name on interface headers. Use GVOS as the product name. GetVirtual can appear only as the company context where needed.

## 26. Important UI Rules

1. Use GVOS as the platform name.
2. Do not make the UI look like a dating, marketplace or ordinary freelancing platform.
3. Do not show talent browsing to clients.
4. Make workspaces the center of the experience.
5. Make task status very visible.
6. Make billing status very visible to clients.
7. Make risk and complaint alerts very visible to managers and admins.
8. Make exact time logs visible only in manager and admin interfaces.
9. Make weekly summaries visible to clients.
10. Make trial restrictions clear for active leads.
11. Make password vault feel secure and controlled.
12. Make mobile responsive design mandatory.

## 28. Final Instruction for Stitch

For every screen, maintain the same GVOS design system. Use consistent sidebar, spacing, typography, colors, cards, badges, tables and buttons.

The UI must be beautiful enough for clients but powerful enough for internal operations.

The goal is not decoration. The goal is clarity, control, trust and productivity.
